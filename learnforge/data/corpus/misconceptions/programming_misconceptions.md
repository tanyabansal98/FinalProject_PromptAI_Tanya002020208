# Programming Misconceptions
## = means equals
In most languages, = is assignment (set value) and == is comparison (check equality). Common bug source.
## Arrays and lists are the same
Arrays have fixed size and type. Python lists are dynamic and heterogeneous. Performance characteristics differ.
## OOP is always better
Functional programming is excellent for data pipelines. Simple scripts don't need classes. Use the right paradigm for the problem.
## Comments explain what code does
Good comments explain WHY, not WHAT. The code itself should be readable enough to show what it does.
